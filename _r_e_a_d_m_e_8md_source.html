1 # QCtrlSignals
2 A library to catch and handle windows and unix signals, in a cross platform manner.
3 
4 With this class, you can easily register for the Operation system signals (like `CTRL_C_EVENT` on windows and `SIGINT` on unix/linux) and use them just like normal Qt signals.
5 
6 ## Features
7 - Register for common signals via a Cross-Platform enum
8 - You can register native signals as well
9 - Asynchronous signal handling. The class emits a Qt signal, and you can connect to this one just like to any other Qt signal
10 - Automated Quit:
11  - The QCtrlSignalHandler automatically registers for typical "quit" signals (like `SIGINT`, `SIGTERN`, ...)
12  - It automatically handles those and then calles `qApp->quit();`
13  - Windows: Can handle the `CTRL_CLOSE_EVENT`, which cannot be catched with the handler otherwise
14 
15 ## Usage
16 Just copy the repository into you application (preferebly by adding it as a git submodule) and add the line `include(./QCtrlSignals/qctrlsignals.pri)` to your .pro-file. This way all files and required libraries will automatically be added. Use `#include <QCtrlSignals>` to access the class.
17 
18 ### Example
19 The following exaple uses the handler to:
20 1. Automatically quit the application (for example by using `CTRL_BREAK_EVENT` or `SIGQUIT`)
21 2. Instead if quitting, `SigInt` (`CTRL_C_EVENT`, `SIGINT`) will print a message
22 
23 ```cpp
24 #include <QCoreApplication>
25 #include <QCtrlSignals>
26 #include <QDebug>
27 
28 int main(int argc, char *argv[])
29 {
30  QCoreApplication a(argc, argv);
31 
32  auto handler = QCtrlSignalHandler::instance();
33 
34  QObject::connect(qApp, &QCoreApplication::aboutToQuit, qApp, [](){
35  qDebug() << "App about to quit!";
36  });
37  handler->setAutoQuitActive(true);
38 
39  qDebug() << "SigInt" << handler->registerForSignal(QCtrlSignalHandler::SigInt);
40  QObject::connect(handler, &QCtrlSignalHandler::sigInt, qApp, [](){
41  qDebug() << "SigInt triggered, shutdown was overwritten for this one!";
42  });
43 
44  return a.exec();
45 }
46 ```

48 ### Logging
49 By default, QCtrlSignals prints some warning messages if something goes wrong (For example, a signal cannot be registered). All messages of QCtrlSignals are grouped into the QLoggingCategory "QCtrlSignals". If you want to simply disable the logging, call the folling function somewhere in your code:
50 ```cpp
51 QLoggingCategory::setFilterRules(QStringLiteral("QCtrlSignals.warning=false"));
52 ```
53 For more information about all the things you can do with the logging categories, check the Qt-Documentation

60 ### Known Limitations
61 - Only such signals, that the operating system allows, can be registered. For example, `SIGKILL` on linux cannot be registered
62 - On Windows, all signals are handeled in a second thread. Thus, the signals that require immediate handling, like `CTRL_CLOSE_EVENT`, cannot be registered
63  - However, this will be handeled for shutdown
